# Scenarios

## 场景 1 — 运营人员手动触发索引

RAG 管理页适合让内部用户查看待索引文档、触发单文档索引或批量索引，并观察进度。

## 场景 2 — 调试搜索质量

RAG Debug 页面可以直接输入查询，选择知识库并查看返回文本、来源、分数和匹配 Chunk。它适合排查 Embedding 模型、切块大小和文档质量问题。

## 场景 3 — 调整切块器路由

RAG Chunkers 页面可以查看扩展名当前使用的切块器，预览路由变更会影响多少文档，再决定是否应用变更并重新索引。

## Common mistakes

- 只看 UI 页面是否加载，却忽略后端向量库连接是否成功；应先使用 RAG 管理页的诊断能力。
- 改了切块器路由但没有重新索引受影响文档。
- 在生产环境使用内存向量库，导致重启后索引数据丢失。
