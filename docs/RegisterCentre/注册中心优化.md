# ZSET 优化注册中心实例发现

## 目标
使用 Redis ZSET（Sorted Set）替代 SCAN 操作来优化 `GetAllInstancesAsync` 和 `GetAllLeaderInstancesAsync` 的性能。

## 背景
- 当前使用 `ScanKeysAsync` 是 O(N) 操作，随实例增加性能下降
- 架构师建议使用 ZSET，Score 存储心跳时间戳，可实现 O(log N) 查询和自动清理过期数据

## 数据结构设计

### 键命名约定
```
现有数据键（保持不变）：
- 实例数据: [KeyPrefix:]reg&&{ServiceName}:{InstanceId} → InstanceState JSON
- Leader数据: [KeyPrefix:]leader&&{ServiceName} → LeaderState JSON

新增索引键：
- 实例索引: [KeyPrefix:]regidx&&{ServiceName} → ZSET (member=InstanceId, score=心跳时间戳)
- Leader索引: [KeyPrefix:]leaderidx → ZSET (member=ServiceName, score=选举时间戳)
```

### ZSET 使用模式
- **ZADD**: 注册/心跳时更新索引，Score = Unix 时间戳
- **ZRANGEBYSCORE**: 获取活跃实例（Score >= 阈值）
- **ZREMRANGEBYSCORE**: 惰性清理过期条目（Score < 阈值）

### Step 3: 重构 RegistrationStateManager

#### 3.1 添加索引常量和辅助方法
```csharp
private const string REG_INDEX_PREFIX = "regidx";
private const string LEADER_INDEX_PREFIX = "leaderidx";

private readonly ISortedSetStateStore? _sortedSetStore = stateStore as ISortedSetStateStore;

private static double ToUnixTimestamp(DateTime utc) =>
    new DateTimeOffset(utc).ToUnixTimeSeconds();
```

#### 3.2 修改 RegisterOrHeartbeatAsync
在保存数据后，同时更新 ZSET 索引：
```csharp
// 保存实例数据（现有逻辑）
await stateStore.SaveStateAsync(regKey, instanceState, REG_PREFIX, ct, ttl);

// 更新 ZSET 索引
if (_sortedSetStore != null)
{
    await _sortedSetStore.SortedSetAddAsync(
        serviceName,      // ZSET key
        regKey,           // member = "ServiceName:InstanceId"
        ToUnixTimestamp(now),
        REG_INDEX_PREFIX,
        ct);
}
```

#### 3.3 重写 GetAllInstancesAsync
```csharp
// 1. 惰性清理过期条目
var threshold = ToUnixTimestamp(DateTime.UtcNow - _option.Election.RegistrationTTL);
await _sortedSetStore.SortedSetRemoveRangeByScoreAsync(
    serviceName, double.NegativeInfinity, threshold, REG_INDEX_PREFIX, ct);

// 2. 获取活跃实例键
var activeMembers = await _sortedSetStore.SortedSetRangeByScoreAsync(
    serviceName, threshold, double.PositiveInfinity, REG_INDEX_PREFIX, ct);

// 3. 批量获取实例数据
var instanceKeys = activeMembers.Select(m => m.Member).ToList();
var instances = await stateStore.GetBulkStateAsync<InstanceState>(instanceKeys, REG_PREFIX, ...);
```

#### 3.4 添加 SCAN 回退逻辑
如果 StateStore 不支持 ZSET，自动回退到原有 SCAN 实现：
```csharp
if (_sortedSetStore != null)
    return await GetAllInstancesWithZSetAsync(ct);
else
    return await GetAllInstancesWithScanAsync(ct);  // 保留原有逻辑
```

## 边界情况处理

1. **数据写入成功，索引写入失败**：
   - 接受最终一致性，下次心跳会修复索引
   - 保留 SCAN 回退确保发现仍然有效

2. **索引有条目但数据已过期**：
   - `GetBulkStateAsync` 返回 null
   - `removeEmptyValue: true` 自动过滤

3. **时钟漂移**：
   - 查询时添加 60 秒缓冲
   - TTL 是过期的权威来源

## 性能对比

| 操作 | SCAN (当前) | ZSET (优化后) |
|------|------------|--------------|
| 心跳写入 | O(1) | O(1) + O(log N) |
| 获取所有实例 | O(N) | O(log N) |
| 索引维护 | 无 | O(log N) 惰性清理 |

## 设计决策（已确认）
- **接口设计**: 独立 `ISortedSetStateStore` 接口，由 `RedisStateStore` 实现
- **Leader 查询**: 保持 SCAN（Leader 数量少，无需优化）
- **清理策略**: 惰性清理（在 `GetAllInstancesAsync` 调用时清理过期条目）
